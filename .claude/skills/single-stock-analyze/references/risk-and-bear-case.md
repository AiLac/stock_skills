# 风险与利空（含红线 / kill-switch）

成长股最怕"高预期 + 一次坏消息"。本文件做风险/利空的系统扫描清单，把每类利空映射到评分卡的惩罚项，并给出三条**红线（kill-switch）**的精确触发条件——命中任一红线，无论原始分多高，结论直接封顶为"回避 / 观望"。筹码面（解禁/增发/内部人卖出/降评级）与期权市场解读单独放在 `references/positioning-and-options.md`；本文件聚焦结构性风险 + 业绩/估值/竞争/客户/监管/会计/宏观利空 + 红线 + 下行情景。

**取证原则**：利空类信息（解禁日、内部人卖出、降评级、会计红旗等）**必须联网查证，不靠记忆**；拿不到的标"待核实"并给查证路径。每条结论标证据分级（强/中/弱/待核实，见 serenity `references/evidence-ladder.md`；兜底：强=SEC 文件/官方数据，中=可信媒体/卖方，弱=社媒/传闻）。

## 风险 vs 利空

先把两个概念分开，否则会把"长期隐患"和"近期坏消息"混为一谈。

- **风险（risk）＝结构性、长期存在的隐患**：竞争格局会不会恶化、单位经济能不能跑通、监管会不会变天、估值能不能撑住。它一直在，不一定近期落地，用来调"质量分"与惩罚项的基线。
- **利空（bear catalyst）＝具体的、可能近期落地的坏消息 / 负面催化**：指引下修、解禁日、增发、内部人集体减持、评级下调、诉讼/监管处罚。它有时间窗（date window），是惩罚项与红线判断的主要触发源。

写报告时：风险进「结构性风险」小节，利空进对应的惩罚项 + 给出 date window；致命级别的进红线。

## 利空类别扫描

逐类扫描，每类映射到一个惩罚项 key（评分卡 `growth_scorecard.py` 中定义，0-5，×2 倍率压低总分）。判分锚点见 `references/growth-scoring-rubric.md`。

- **业绩 / 指引利空** → 这是增长引擎是否还在的核心信号。增速放缓、指引下修、不及预期。轻则压低增长因子，重则触发红线 `growth_engine_broken`（见下）；指引相关深读见 `references/earnings-readthrough.md`。
- **估值杀** → `liquidity_bubble`（惩罚项）。利率/风险偏好逆转时，高估值最先挨杀；泡沫化估值 + 流动性差时计入此项。纯"贵"主要压低买点因子（`valuation_growth_match` / `entry_expectations`，见 `references/valuation-and-entry.md`），泡沫+流动性差才升级为惩罚。
- **竞争与技术颠覆** → `competition_disruption`（惩罚项）。新进入者、替代技术、平台方自研、价格战，侵蚀份额或定价权。
- **客户流失 / 集中度** → `customer_concentration`（惩罚项）。单一大客户占比过高、流失风险高。注意区分：**集中度高**计惩罚项；**核心客户已确认流失**升级为红线 `core_customer_loss`（见下）。
- **监管 / 地缘 / 政策** → `regulation_geopolitics`（惩罚项）。出口管制、制裁、反垄断、数据/隐私、关税、军事终端用途、当地审批等。
- **会计与治理红旗** → `accounting_quality`（惩罚项，账务质量）+ `governance`（惩罚项，治理）。会计：SBC 占收比畸高、billings/收入背离、存货/应收 vs 收入背离、一次性项目虚增、激进收入确认。治理：关联交易、双重股权下的控制人滥权、频繁更换审计/CFO、补发/重述。**重大造假嫌疑**升级为红线 `accounting_fraud_suspicion`（见下）。
- **宏观 / 流动性** → `liquidity_bubble`（惩罚项）。宏观利率、风险偏好、个股流动性差（微盘、成交稀薄）放大波动与下行。
- **筹码面利空（解禁/增发/内部人卖出/降评级）** → `dilution_financing` + `lockup_insider_supply`（惩罚项）。属供给/筹码面，**完整清单、判分与数据源见 `references/positioning-and-options.md`**，此处不重复。

## 红线（kill-switch）标准

红线是"致命利空"——命中即把结论封顶为"回避 / 观望（红线封顶）"，不让高分掩盖致命问题。评分卡 `growth_scorecard.py` 中定义三个布尔红线 key；任一为 true 即封顶（`capped_by_red_line=true`）。判定从严：证据不足时**不要**轻易置 true，而是标"待核实"并在报告中说明查证路径。

- **`accounting_fraud_suspicion`（重大财务造假嫌疑）**：触发条件——出现以下任一**强证据**：做空机构有实据指控且公司无法有力反驳、SEC/监管正式立案调查财务问题、审计师出具保留/否定意见或拒绝签字、被迫重述（restatement）核心财务报表、CFO/审计因财务争议突然离职且伴随其他红旗。仅"会计指标偏激进"不够（那是 `accounting_quality` 惩罚项），需上升到"数字可能是假的"层级。
- **`core_customer_loss`（核心客户流失）**：触发条件——占收入重大比例（如单一客户/前几大客户合计达高占比）的核心客户**已确认**流失、转单、自研替代或大幅砍单，且无可见的等量替代。仅"客户集中度高"不够（那是 `customer_concentration` 惩罚项），需是"已经发生、伤及收入根基"。
- **`growth_engine_broken`（增长引擎失效）**：触发条件——增长叙事的核心驱动**已被证伪或熄火**：核心产品需求断崖、连续多季增速塌方且管理层下修全年指引、TAM 见顶/被颠覆、净留存跌破荣枯线且持续。即"成长股不再成长"。单季 miss 不算，需结构性、可持续的失速 + 指引印证。

封顶机制本身见评分卡：命中红线时 `verdict` 置为 `回避 / 观望（红线封顶）`，与原始分脱钩。

## 下行情景

给结论加一个"如果错了能跌多少"的量级感，而不只说"有风险"。

- **逻辑层**：先写"最可能让判断翻车的那条链"——通常是增长引擎熄火或估值杀，对应上面的利空/红线。把它写成一句可证伪的话（"若下季指引再下修，则 …"）。
- **估值底层**：用更保守的增速/利润率假设，或回到更低的历史估值分位（如 EV/NTM Sales 历史低位）反推一个**估值地板**，给出"悲观情景下的价格量级"。明确标注为"基于假设的测算"而非硬数据（数据获取见 `references/valuation-and-entry.md`）。
- **触发器与时间窗**：把下行情景挂到具体催化（下次财报、解禁日、监管裁决）上，给 date window，方便用户盯。

## 复用 serenity

风险合规边界沿用 serenity `references/risk-and-compliance.md`（研究判断、不替用户下单；不用"保证收益/必涨"措辞；micro-cap/社媒驱动/单一未具名客户/重融资稀释等高风险情形需额外谨慎）。**兜底**：serenity 不在场时，SKILL.md 内含精简版合规说明——结论是研究信心（值得投/观望/回避），交易决定在用户；绝不编造价格/财报数字/来源，拿不到标"待核实"并给查证路径。
