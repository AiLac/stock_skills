# 最近一次财报深读 checklist

最新财报是判断成长股**最新鲜、最硬**的证据：增速有没有加速、指引上不上调、产品有没有放量，都在这里见分晓。给定一个美股代码后，把最近一份季报/年报当作核心证据逐项扫一遍，把结论填进评分卡与报告「📊 最近财报重点」节。

**来源（强证据）**：最新 10-Q / 10-K（SEC EDGAR）+ 财报电话会 transcript + 业绩 PPT / investor presentation；现价、beat/miss 幅度、分析师预期修正等**必须联网查证，不靠记忆**，拿不到的标"待核实"并给查证路径。证据分级见 serenity `references/evidence-ladder.md`（兜底：强=文件/transcript/PPT，中=可信媒体/卖方，弱=社媒/传闻）。

**评分联动（贯穿全篇）**：指引上调 / 增速加速 → `revenue_growth_durability` 等质量因子**加分**；指引下修、撤回中长期模型、核心增长业务由增转降 → 触发 `growth_engine_broken` **红线判断** + 计入相应惩罚项（如 `liquidity_bubble` 估值杀、`accounting_quality` 红旗）。每条结论标 强/中/弱/待核实。判分锚点见 `references/growth-scoring-rubric.md`。

**目录（八大块，按重要性排序）**
1. [增长实绩](#1-增长实绩) → `revenue_growth_durability`
2. [指引](#2-指引成长股核心) → `revenue_growth_durability` / `growth_engine_broken`
3. [核心产品与研发进展](#3-核心产品与研发进展) → `product_customer_momentum`
4. [盈利与单位经济性](#4-盈利与单位经济性) → `unit_economics_profit_path`
5. [客户与势能](#5-客户与势能) → `product_customer_momentum`

7. [电话会弦外之音与红旗](#7-电话会弦外之音与红旗) → `accounting_quality` / `governance`

## 1. 增长实绩

抓本季实际增长的"质"，不只看表面数字。

- 实际收入 vs 一致预期：beat / miss 及**幅度**（多少 %）。
- YoY 与 QoQ 增速；趋势是**加速还是放缓**（连续 2-3 个季度看方向）。
- 按业务线 / 产品 / 地区拆收入：哪一块在拉动增长、哪一块在拖累。
- 增长的"质"：是真实需求，还是一次性 / 拉动透支（pull-forward）/ 并表带来。
- → 写入 `revenue_growth_durability`（加速且可持续=高分；放缓或靠一次性=低分）。

## 2. 指引（成长股核心）

指引是成长股最关键的前瞻信号，比当季数字更影响结论。

- 下季度收入 / 利润指引 vs 市场预期：高于 / 符合 / 低于。
- **全年指引动作：上调 / 维持 / 下修**（这是评分联动的核心扳机）。
- 中长期模型 / 目标是否重申、上修，还是改动甚至**撤回**。
- 管理层对需求环境的措辞：更乐观还是更谨慎、有无"能见度下降"等暗示。
- → 指引上调 / 中长期模型上修 → `revenue_growth_durability` **加分**；指引大幅下修 + 撤回中长期模型 → 触发 `growth_engine_broken` **红线判断**，并计入惩罚。

## 3. 核心产品与研发进展

成长股靠新引擎驱动，看产品和研发兑现到哪一步。

- 新品发布与路线图进度：是否按时、是否领先竞品。
- 新品放量节奏（ramp）：处于导入、爬坡还是规模放量阶段。
- 关键技术里程碑：芯片流片 / 临床分期 / 模型版本发布 / 新产能投产等。
- R&D 投入方向与强度：是否押注未来增长引擎。
- 相对竞品的定位变化：领先扩大还是被追近。
- → 写入 `product_customer_momentum`（新品强势放量、路线图领先=高分）。

## 4. 盈利与单位经济性

成长股看"未来能不能赚钱"，单位经济性要往好的方向走。

- 毛利率趋势：扩张 / 稳定 / 恶化。
- 经营杠杆：收入增长时营业利润率是否同步改善。
- EPS（GAAP / non-GAAP）vs 预期；两者差距由什么造成（多为 SBC）。
- SBC 占收入比：是否过高、是否在稀释股东。
- FCF / 现金跑道：是否转正、自我造血能力、烧钱速度与剩余跑道。
- → 写入 `unit_economics_profit_path`（高毛利+经营杠杆+FCF 充沛=高分）；SBC/billings 背离计入 `accounting_quality` 惩罚。

## 5. 客户与势能

势能指标揭示增长的持续性与黏性。

- 净留存（NRR / NDR）：>120% 强、约 100-110% 一般、<100% 警示。
- 大客户数、客户总数增长、客户流失（churn）。
- 订单 / RPO / backlog / billings：是否与收入同步走强，还是背离。
- 客户结构：是否过度依赖单一 / 少数大客户（联动 `customer_concentration` 惩罚）。
- → 写入 `product_customer_momentum`（势能强劲、留存高=高分）。

## 6. 资本配置

管理层怎么花钱，决定增长的含金量与稀释程度。

- 回购 vs 稀释：股本净减少，还是被 SBC / 增发抵消甚至放大。
- 并购：是否创造价值、价格是否合理、整合风险。
- Capex 计划：尤其 AI / 算力等重资本开支的规模与回报路径。
- 现金运用是否与股东利益一致、是否有纪律。
- → 写入 `management_capital_allocation`（纪律强、ROIC 高=高分）；持续高稀释 / 增发在即 → 计入 `dilution_financing` 惩罚。

## 7. 电话会弦外之音与红旗

transcript 的语气与措辞，常比数字更早预警。

- 管理层语气 / 叙事是否变化：从进攻转防守、从具体转含糊。
- 对分析师提问是否**含糊 / 回避 / 答非所问**。
- 一次性项目虚增收入 / 利润的痕迹。
- 存货、应收账款 vs 收入是否背离（增长快于收入=红旗）。
- 需求前置（pull-forward）预警：是否提示透支未来需求。
- → 红旗计入 `accounting_quality` / `governance` 惩罚；若出现重大造假硬信号（重述 / 审计师异动 / 监管立案）→ 触发 `accounting_fraud_suspicion` 红线（需强证据）。

## 8. 财报后反应

市场对财报的真实定价，校准买点与预期面。

- 股价反应：盘后 / 次日涨跌幅，以及与 beat/miss 是否匹配（好财报却大跌=预期过高）。
- 分析师预期修正方向：上修潮还是下修潮、目标价变化。
- 财报前隐含波动（expected move）兑现情况，期权市场怎么解读（详见 `references/positioning-and-options.md`，属中/弱证据）。
- 是否触发降评级 / 上调评级（联动 `lockup_insider_supply` 筹码面惩罚）。
- → 写入 `entry_expectations`（利好出尽 / 预期过高=低分；预期被 reset / 过度悲观=高分）。

## 催化 / 日历事件类型（排进报告第 10 节）

读完财报后，把未来 3-12 个月所有**会让市场重新定价**的事件排成时间线，写进报告第 10 节。每条标**日期或日期窗口**，未知则标"待核实"并给查证路径（公司 IR 日历、SEC、交易所、分析师覆盖）。别只写"下次财报"——逐类过一遍，把适用的都排进去：

- **业绩节奏**：下次财报日期；业绩预告 / preannounce 窗口；指引兑现检验点。
- **产品 / 技术**：重大产品发布、量产、上市（新芯片、新车型、新药获批 / PDUFA、新版本）；关键里程碑（流片、临床读数、产能投产）。
- **公司活动**：行业大会 / 开发者大会 / 投资者日（如 GTC、CES、WWDC、Computex、Analyst Day）。
- **筹码 / 资金面**：解禁日（分批，逐批日期与规模）；增发 / ATM / S-3 生效；回购计划窗口；重大锁定期到期。
- **上下游 / 大客户**：核心客户与上下游的财报及**资本开支指引**（如算力链看 MSFT / GOOGL / AMZN / META 的 capex；设备 / 材料链看晶圆厂 capex 与稼动率）——往往比公司自身财报更早预示需求拐点。
- **监管 / 政策**：出口管制裁决、反垄断 / 诉讼节点、关税、FDA / 监管审批、牌照发放。
- **竞争格局**：竞品发布、价格战、份额数据披露。
- **资金面信号**：分析师评级日 / 目标价调整 / 卖方大会；指数纳入或剔除（如纳入 S&P 500、季度再平衡）。
- **宏观**：FOMC、CPI、长端利率走向（高估值成长股对利率尤其敏感）。
- **公司治理 / 重大事项**：股东大会、重大投票（如薪酬方案）、大额合同 / 订单公告窗口、并购交割。

> 把这些与第 8 步「风险与利空」里的解禁 / 利空日期合并成**一条时间线**，让用户一眼看到"接下来什么时候可能发生什么"。
