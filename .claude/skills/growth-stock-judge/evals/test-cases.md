# 测试用例（growth-stock-judge）

人读的触发 / 行为 / 编排测试清单。可运行版本见 `evals/evals.json`。
逐条核对：给一个美股代码后，skill 应给出「结论先行」的成长股投资判断；纯赛道扫描应让位给 `serenity-skill`。

## 触发测试

应激活本 skill（单股投资判断）：

- [ ] 给一个美股代码并问值不值得投，例如「NVDA 值得投资吗」。
- [ ] 中文口语化提问，例如「帮我分析特斯拉现在能不能买」「判断这只美股值不值得投」。
- [ ] 英文提问，例如「should I buy NVDA」「is TSLA worth investing」「analyze this US stock」。
- [ ] 即使用户没说「分析」，只要给的是单只美股 + 想买/想投的意图，也应触发。

应让位给 serenity（不该硬出单股结论）：

- [ ] 多股赛道扫描 / 排名，例如「美股 AI 半导体这个赛道，哪几家最值得研究」→ 交给 `serenity-skill`，或反问要判断哪一只具体股票。

## 行为测试

输出报告应满足：

- [ ] 结论先行：开头给明确结论（值得投资 / 观望 / 回避，或 强信心值得投资）+ 信心等级。
- [ ] 评分卡在场：8 个因子评分 + 权重，并给出双轴（质量分 + 买点分）。
- [ ] 最近财报节在场：覆盖增速实绩 + 指引（成长股核心）。
- [ ] 风险与利空节在场：含「资金面」（解禁 / 增发 / 内部人卖出 / 降评级）小节 + 「期权市场怎么看」小框。
- [ ] 证据有分级：关键结论标 强 / 中 / 弱 / 待核实。
- [ ] 双轴判断细腻：好公司但偏贵 → 输出「观望 / 分批」这类（质量分高、买点分低）的结论。
- [ ] 命中致命利空时封顶：触发红线（重大财务造假嫌疑 / 核心客户流失 / 增长引擎失效）→ 结论封顶为「回避 / 观望」，高分不掩盖致命问题。
- [ ] 不编数字：现价 / 财报数字 / 来源必须联网查证，拿不到标「待核实」并给查证路径，绝不把编造数字当真实结论。
- [ ] 语言跟随用户：英文提问用英文回答，中文提问用中文回答。
- [ ] 合规边界：研究判断，不替用户下单；不用「保证收益 / 必涨」措辞。

## 编排测试

多 agent 执行模式（fan-out → synthesize）应满足：

- [ ] 7 个研究子代理并行分派（互相独立、无共享状态），各自联网取证。
- [ ] 子代理统一回传 `assets/agent-findings-schema.json` 结构（`findings[]` / `flags[]` / `needs_checking[]` / `suggested_factor_scores` / `suggested_penalties`）。
- [ ] `judge-synthesizer` 在所有研究代理返回后（barrier）汇总，跑 `scripts/growth_scorecard.py` 打分出结论。
- [ ] 单个子代理失败 / 数据不足时，对应模块标「待核实」，不阻塞出结论；命中红线则封顶。
- [ ] 无多 agent 能力时，单 agent 顺序执行兜底仍可跑通。
