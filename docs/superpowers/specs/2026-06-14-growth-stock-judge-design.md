# growth-stock-judge 设计文档

- 日期：2026-06-14
- 状态：已确认，待写实现计划
- 作者：与用户协作（brainstorming）

## 1. 目标与范围

做一个**美股单股投资判断 skill**：用户给一个美股代码（如 NVDA），skill 输出一份完整分析并给出**明确结论——是否值得投资**。

- **核心用法**：单股深度判断（不是赛道筛选/排名）。
- **投资视角**：以**成长投资**为主——重增速、市场空间（TAM）、增长持续性、产品/客户势能，愿为高增长付溢价。
- **结论口径**：结论同时包含「公司质量」和「当前买点（价格/估值贵不贵）」两个维度。
- **数据来源**：以**联网搜索 / deep-research** 为主（SEC 文件、财报电话会 transcript、业绩 PPT、可信媒体、公开数据），**不硬依赖**任何特定 MCP。若会话中恰好有券商/行情类 MCP，可作为可选加速器，但不是前置条件。
- **输出语言**：默认中文（跟随用户语言）。
- **风险边界**：研究判断，不替用户下单（详见第 9 节）。

### 非目标（YAGNI）
- 不做多股筛选与排名（这部分让位给已有的 `serenity-skill`）。
- 不做交易执行、下单、仓位管理。
- 不做组合层面的资产配置。

## 2. 与 serenity-skill 的分工

项目里已有 `serenity-skill`（供应链卡点猎手：赛道扫描、按产业链位置给公司排名，多市场，研究方法论）。

- **serenity**：找"赛道里哪一层卡脖子、谁离稀缺层最近"——*where*。
- **growth-stock-judge（本 skill）**：判"这只具体的美股，现在该不该投"——*is this name worth it now*。

两者形成搭档。本 skill **复用 serenity 的公共资产（引用，不复制）**：
- 证据分级：`serenity-skill/references/evidence-ladder.md`
- 美股数据源路径：`serenity-skill/references/market-source-playbook.md`（US 段）
- 风险合规边界：`serenity-skill/references/risk-and-compliance.md`
- 护城河/产业链位置判断借用 serenity 的"稀缺层/卡点"思路。

SKILL.md 中写明：**若 serenity 不在场，本 skill 自带精简版兜底说明**，保证可独立运行。

## 4. 单股研究流程

给定一个美股代码后，skill 按此顺序研究：

1. **锁定公司**：业务、收入结构、所处赛道与产业链位置。
2. **最近一次财报深读**（独立重点模块，见第 6 节）——靠前，因为这是最新鲜最硬的证据。
3. **增长引擎**：收入增速及趋势、驱动因素、增长可持续性、客户/产品势能（净留存、订单、指引上调）。
4. **市场空间**：TAM 与渗透率、还有多长跑道。
5. **护城河 / 产业链位置**：借 serenity 的"卡点/稀缺层"思路判断竞争壁垒。
6. **单位经济性**：毛利、经营杠杆、FCF 趋势、盈利路径（成长股看"未来能不能赚钱"）。
7. **估值与买点**：P/S、PEG 类、对比历史/同业、隐含预期；现价是否合理。
8. **风险与利空扫描**（独立模块，见第 7 节）：结构性风险 + 业绩/估值利空 + 筹码面（解禁/增发/内部人卖出/降评级）+ 期权市场解读。
9. **证据分级 + 反方**：每个关键结论标 强/中/弱/待核实；列"什么情况说明判断错了"。
10. **打分出结论**（评分卡，见第 5 节）。

现价、现状、最新财报等**必须联网查证，不靠记忆**；拿不到的数据标"待核实"并给查证路径。

## 5. 评分卡设计

仿 `serenity_scorecard.py`：每个因子 0-5 分 × 权重（合计 100），再减惩罚项（×2 倍率），最后红线封顶。

### 5.1 因子与权重（成长投资）

| 因子 | 权重 |
|---|---:|
| 收入增速与持续性 | 18 |
| 市场空间 / 渗透率（TAM） | 14 |
| 护城河 / 产业链位置 | 14 |
| 单位经济性 / 盈利路径 | 14 |
| 产品与客户势能 | 12 |
| 管理层与资本配置 | 8 |
| **估值与增长匹配度** | 12 |
| **买点 / 预期面** | 8 |

加粗两项 = 「当前买点」共 20 分：价格能影响结论，但不会单凭"贵"就否掉一家好公司。

### 5.2 惩罚项（×2 倍率，实打实压低总分）
稀释/融资、客户集中、会计质量（SBC/billings 背离）、治理、竞争颠覆、监管/地缘、流动性/泡沫、解禁/增发/内部人卖出/机构降评级（筹码面利空）。

### 5.3 红线（Kill-switch）封顶机制
若命中致命利空——如重大财务造假嫌疑、核心客户流失、增长引擎失效——**直接将结论封顶为"回避/观望"**，不让高分掩盖致命问题。

### 5.4 双轴结论
除总分外，另算：
- **质量分**：前 6 项因子（增速、TAM、护城河、单位经济、势能、管理层）。
- **买点分**：后 2 项因子（估值匹配度、买点/预期面）。

于是能输出"好公司但偏贵 → 观望/分批"这类细腻判断。

### 5.5 结论分档
- ≥85：强信心值得投资
- 70–84：值得投资
- 55–69：观望 / 建仓试探
- <55：暂不值得 / 回避
- 命中红线：封顶为"回避 / 观望"（无论原始分多高）

## 6. 最近一次财报深读模块

来源：最新 10-Q/10-K + 财报电话会 transcript + 业绩 PPT（强证据）。`references/earnings-readthrough.md` 做成逐项 checklist。

1. **增长实绩**：实际收入 vs 一致预期（beat/miss 幅度）；YoY/QoQ 增速及**加速 or 放缓**；分业务线/产品/地区拆收入，看哪块拉动增长。
2. **指引（成长股核心）**：下季度指引 vs 预期；**全年收入/利润指引：上调/维持/下修**；中长期模型是否重申/改动；管理层对需求环境的措辞。
3. **核心产品与研发进展**：新品发布与路线图进度、新品放量节奏（ramp）；关键技术里程碑（芯片流片、临床分期、模型/版本发布、产能投产）；R&D 投入方向；相对竞品定位变化。
4. **盈利与单位经济性**：毛利率趋势、经营杠杆、营业利润率；EPS（GAAP/non-GAAP）vs 预期；SBC 占收比；FCF / 现金跑道。
5. **客户与势能**：净留存（NRR/NDR）、大客户数、客户数增长、流失；订单/RPO/backlog/billings。
6. **资本配置**：回购/稀释、并购、Capex 计划（尤其 AI/算力资本开支）。
7. **电话会弦外之音与红旗**：管理层语气/叙事变化；对分析师提问是否含糊/回避；一次性项目虚增；存货/应收 vs 收入背离；需求前置（pull-forward）预警。
8. **财报后反应**：股价反应；分析师预期修正方向（上修/下修潮）。

联动：指引上调 → 增速因子加分；指引下修 → 利空 + 惩罚。

## 7. 风险与利空模块

区分两个概念：
- **风险**＝结构性、长期存在的隐患。
- **利空**＝具体的、可能近期落地的坏消息/负面催化。

### 7.1 流程层扫描类别
- 业绩/指引利空：增速放缓、指引下修、不及预期。
- 估值杀：杀估值、利率/风险偏好逆转（高估值最怕）。
- 竞争与技术颠覆、客户流失/集中度。
- **资金面/筹码面**（见 7.2）。
- 监管/地缘/政策；会计与治理红旗；宏观/流动性。

### 7.2 资金面 / 筹码面利空
- **解禁日（lockup expiry）**：日期、规模（占流通盘 %）、谁解禁。
- **增发 / ATM**：shelf 注册（S-3）、ATM 计划、配股——稀释风险。
- **内部人卖出（insider selling）**：Form 4、是否 10b5-1、是否高管集体减持（CEO/CFO 集中减持信号更强）。
- **机构降评级 / 减持**：分析师评级下调、目标价下修潮；13F 机构持仓下降。
- **回购 vs 稀释净额**：股本真减少还是被 SBC 抵消。

### 7.3 期权市场解读
- 隐含波动率 IV + IV 百分位：市场预期波动多大、当前贵不贵。
- 财报隐含波动（expected move）：期权 price-in 的下次财报跳空幅度。
- Put/Call 比率：多空情绪。
- 偏斜 skew：下行保护需求（put skew 走高＝避险/恐慌）。
- 未平仓 OI 集中度 / max pain：关键行权价、磁吸位。
- 异常期权活动：大单/不寻常活跃（仅作 lead 级线索）。
- 期限结构：催化剂前的近月升水/倒挂。

### 7.4 证据分级
期权与资金面信号属**情绪/持仓面**，定为**中/弱证据**，用来判断"市场在 price 什么"、辅助买点与风险，**不作为核心结论主驱动**。部分期权数据（IV 百分位、skew）联网不一定拿得全——拿不到就标"待核实"并给查证路径（EDGAR Form 4 / S-3、交易所解禁日历、期权数据源）。

### 7.5 三层落地
- **流程层**：风险与利空扫描下设 *资金面/筹码面 + 期权* 子步骤。
- **评分层**：解禁/增发/内部人卖出/降评级 → 利空惩罚项；财报前高 IV + put skew → 压低买点因子；致命利空 → 红线封顶。
- **输出层**：报告「⚠️ 风险与利空」节内含资金面小节 + 「期权市场怎么看」小框 + 下行情景 + kill-switch 清单。

## 8. 输出报告结构（结论先行 → 评分卡 → 详情）

1. **结论框**：一句话结论 + 信心等级 + **双轴**（质量分/买点分）+ 总分。例：*"好公司，但当前偏贵 → 观望/分批；质量 82 / 买点 48 / 总分 71"*。
2. **评分卡表**：各因子得分 + 惩罚项 + 红线状态（是否触发封顶）。
3. **投资逻辑**（成长故事一段话）。
4. **增长引擎 + 市场空间**。
5. **📊 最近财报重点**（增速/指引/研发产品进展/红旗/财报后反应）。
6. **护城河 / 产业链位置**。
7. **单位经济性 / 盈利路径**。
8. **估值与买点**。
9. **⚠️ 风险与利空**：结构性风险 → 资金面（解禁/增发/内部人卖出/降评级）→ 「期权市场怎么看」小框 → 下行情景 → kill-switch 清单。
10. **结论复述 + 下一步要核实的清单 + 催化/日历**（下次财报、解禁日等）。
11. **证据与来源**（每条标 强/中/弱/待核实）。

## 9. 风险合规与语言

- **研究判断，不替用户下单**：结论是研究信心（值得投/观望/回避），交易决定在用户；不用"保证收益/必涨"措辞；沿用 serenity 的 `risk-and-compliance.md`。
- **绝不编造**价格、财报数字、来源；拿不到就标"待核实"并给查证路径；现价/现状必须联网查，不靠记忆。
- **语言**：默认中文（跟随用户语言）；结论先行；大白话，少术语。

## 10. 测试用例（`evals/test-cases.md`）

- **触发测试**：给美股代码、"NVDA 值得投资吗""帮我分析特斯拉现在能不能买" → 应激活本 skill；纯赛道扫描 → 让位给 serenity。
- **行为测试**：结论先行✓、评分卡在✓、最近财报节在✓、风险与利空节在（含资金面+期权）✓、证据有分级✓、命中致命利空时封顶✓、不编数字✓。
- **走查示例**：可选 `examples/<ticker>-demo.md`，拿一只成长股完整跑一遍展示形态。
